Throughout the Computer Science program, I have had the opportunity to work on a variety of projects that helped me develop both technical and professional skills. Looking back at the work I completed, I can see how much my understanding of software development has grown. While early projects focused on learning individual concepts and programming techniques, later courses required me to combine those skills to create more complete and practical software solutions.

Creating this ePortfolio gave me the chance to revisit some of my previous work and think about how the different courses connected together. It also allowed me to showcase the areas of computer science that I feel most comfortable with, including software development and problem-solving. The portfolio demonstrates not only the work I completed, but also how I can take an existing project and improve it by applying new skills and techniques learned throughout the program.

One skill that was reinforced throughout the program was communication. In many courses, projects required documentation, planning, and explanations of technical decisions. Being able to explain why a solution was chosen became just as important as getting the solution to work. This helped me become more comfortable communicating technical information in a way that others can understand. If I went back to something that I hadn't worked on in a while. I wanted to be able to understand it again with short glances at the documentation. The same idea goes for sharing my work with others. 

The program also helped me understand the importance of collaboration. Even though most assignments were completed individually, there were opportunities to review the work of others, participate in discussions, and consider different approaches to solving the same problem. These experiences showed me that software development is rarely about finding a single correct answer. Instead, it often involves comparing different solutions and working toward the one that best meets the requirements of a project.

Data structures and algorithms were another important part of the program. As projects became more complex, I learned that organizing and processing data efficiently can have a major impact on how useful an application becomes. Learning how to approach problems logically and develop algorithms to solve them strengthened my problem-solving skills and helped me think more critically about the code I write.

Software engineering concepts were present throughout nearly every course. From planning and designing applications to testing and maintaining them, I learned that writing code is only one part of the development process. Creating software that is organized, maintainable, and easy to improve later is just as important. These ideas became especially important during the capstone project, where I revisited an existing application and focused on making meaningful improvements rather than simply adding new features.

Database development also played a major role in my coursework. Working with SQL databases helped me understand how applications store, retrieve, and manage information. I gained a better appreciation for how important a well-designed database can be to the overall success of an application. Managing data efficiently and ensuring that information remains accurate will continue to be important in future software development projects.

Security was another topic that appeared throughout the program. While different courses approached it from different angles, they all emphasized the importance of protecting information and designing systems responsibly. Whether working with user accounts, application data, or database access, I learned that security should be considered throughout development rather than added afterward. Security is something that, looking back, could be easily missed from an outside perspective. I never really knew how much it's thought of in the development and design phases of a program or software. 

The artifact selected for this portfolio is my "Flux" Weight Tracking application, which was originally developed in Android Studio. I chose this project because it was one of the assignments I enjoyed the most and because I felt there were several areas that could be improved. The enhancements focus on software design and engineering, algorithms and data structures, and databases. 

The software design enhancement improves the overall appearance and organization of the application through the use of Material Design components and a cleaner interface. The algorithms and data structures enhancement introduces features that calculate average weight and identify weight trends, giving users more meaningful information from their stored data. This, however, was later changed to a more meaningful structure of showing off the average, the highest, the lowest, and the number of entries, as I found this a more visually appealing and information-driven approach to updating the data structure and some of the internal algorithms. The database enhancement improves how information is managed within the application by simplifying data handling and allowing users to edit and delete entries with live updates. The live updates were a huge part, because previously, all updates were not live - users had to log out and then log back into the application to see anything. 

Together, these artifacts demonstrate the different skills I have developed throughout the Computer Science program. While each enhancement focuses on a different area of computer science, they all contribute to the same goal of creating a more useful and maintainable application. This portfolio serves as a representation of the knowledge and experience I have gained throughout the program and demonstrates my ability to apply those skills to improve existing software solutions.
